为什么会有Java内存模型

Java实现“一次编写，到处运行”的特性就是依赖JVM，JVM只要对class文件负责就好，无论你的源代码是Java还是Kotlin只要编译成为符合JVM规范的class文件就可以了。

JVM是一个虚拟机，有自己的指令集和内存管理，不同的操作系统和底层硬件对于内存的操作可能存在差异，内存中存放的就是运行时的数据，如果因为差异导致数据错误是不可以的。

Java内存模型

JVM规范将运行时内存分成如下:

  • 线程私有:
    • 程序计数器
    • 本地方法栈
    • 虚拟机栈
  • 线程共有:
    • 方法区
    • 堆
名称 线程私有 作用
程序计数器 是 记录代码执行的位置
本地方法栈 是 记录native方法
虚拟机栈 是 函数压栈、局部变量
方法区 否
堆 否 对象存放的地方
